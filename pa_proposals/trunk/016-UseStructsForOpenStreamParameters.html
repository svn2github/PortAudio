Proposed Enhancements to PortAudio API

016 - Use Structs for Pa_OpenStream() Parameters

Enhancement Proposals Index, PortAudio Home Page

Updated: August 20, 2002

Status

This proposal is under discussion.

Background

The current version of Pa_OpenStream() has 16 parameters. This is a very large number of parameters for a C function to take. Code that calls Pa_OpenStream may be difficult to read, as it is difficult to identify which parameters are which.

Proposal

Define a new structure called PaDeviceParameters: 

typedef struct PaDeviceParameters{
     PaDeviceIndex device;
     int numChannels;
     PaSampleFormat sampleFormat;
     PaTime latency;
     void *hostApiSpecificStreamInfo;
}PaDeviceParameters;

Replace existing input and output device parameter fields in Pa_OpenStream() with pointers to PaDeviceParameters structures. 

 PaError  Pa_OpenStream (
   PaStream **stream,
   PaStreamParameters *inputParameters,
   PaStreamParameters *outputParameters,
   double sampleRate,
   unsigned long framesPerBuffer,
   PaStreamFlags streamFlags,
   PortAudioCallback *callback,
   void *userData);

The same could be done for Pa_IsFormatSupported(): 

Pa_IsFormatSupported(
   PaStreamParameters *inputParameters,
   PaStreamParameters *outputParameters,
   double sampleRate );

Half duplex streams should pass NULL for the parameter structure which they are not using.

Discussion

Using structures to pack function parameters has the advantage of creating more readable code, because clients will generally assign values to these structures as follows: 

PaStreamParameters inputParameters;
inputParameters.device = Pa_GetDefaultInputDevice();
inputParameters.numChannels = 2;
inputParameters.sampleFormat = paFloat32;
inputParameters.latency = Pa_GetDefaultLowInputLatency( inputParameters.device );
inputParameters.hostApiSpecificStreamInfo = NULL;

Impact Analysis

This proposal will require clients that use Pa_OpenStream() to rewrite the parameter passing code.
