Proposed Enhancements to PortAudio API

015 - Improve Callback Timestamp Info

Enhancement Proposals Index, PortAudio Home Page

Updated: July 23, 2002

Status

This proposal is under construction.

Background

The information provided by the outTime callback parameter and Pa_StreamTime() function does not provide sufficient information to perform some syncronisation tasks such as syncronising PortAudio generated audio with another timebase (such as syncronisation pulses in a timestamped MIDI event stream.)

Proposal

Remove Pa_StreamTime(). Provide a common time base ( Pa_GetTime() ) and pass three time parameters to the callback: current time, time at which the first sample of the input buffer was recieved, time at which the first sample of the output buffer will play, all relative to the Pa_GetTime() timebase.

Impact Analysis

This proposal will require clients to rewrite code that depends on the Pa_StreamTime() parameter.
